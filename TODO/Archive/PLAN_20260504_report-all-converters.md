## Goal

1. Reshape `tools/generate_report.py` from "convert one user-supplied vendor file" into "convert one canonical example **per implemented converter** and emit an index page with one row per converter."
2. Introduce `loguru` as the package's logging backbone and migrate every existing `print()` call.
3. Capture per-converter run output to a sidecar log file linked from the index.

## User decisions (locked)

- **Clear output dir at start of a full run:** yes.
- **Index columns:** 6 — `software | input | output | dim/layer | report | log`.
- **Show skipped/failed rows:** yes; include log column for them too.
- **Logging library:** `loguru`.
- **Migration scope:** all 11 `print()` calls migrate. No "stdout-as-data" exceptions — there is no concrete consumer piping any of this output, so the distinction was a fiction.

## Index columns (6)

| software | input | output (.h5ad) | dim / layer | report | log |

- **software** — `rule.software_name` from the matched packaged rule.
- **input** — full path to the source vendor file.
- **output** — link to `<software>_<sha8>.h5ad`. For skipped/failed rows: "(none)".
- **dim / layer** — `(n_obs, n_var)` and a `<ul>` of `<li><code>{layer}</code>: ({rows}, {cols})</li>` per layer. Empty for skipped/failed rows.
- **report** — link to `<software>_<sha8>.html`. "(none)" for skipped/failed rows.
- **log** — link to `<software>_<sha8>.log`. Always present (the log captures the skipped/failed reason too).

One row per packaged `ParseRule` (currently 6: DIA-NN, Spectronaut, MaxQuant, FragPipe, PEAKS, WOMBAT).

## Source of truth for inputs

The repo already has the canonical mapping that the test suite uses:

- Cache directory: `test_data_download/json_dir/` (gitignored, regenerated by `test_data_download/Makefile`)
- Index: `test_data_download/raw_file_db_downloaded.csv` — columns include `software_name`, `input_file_path`, `status`
- Existing lookup pattern (used by [tests/test_converters_e2e.py](../tests/test_converters_e2e.py:18-26) and [tests/test_readers_integration.py](../tests/test_readers_integration.py:18-26)):

```python
def find_test_data(software_name: str) -> Path | None:
    if not DOWNLOADED_DB.exists():
        return None
    with open(DOWNLOADED_DB) as f:
        for row in csv.DictReader(f):
            if row["software_name"] == software_name and row.get("status") == "ok":
                return TEST_DATA_DIR / "json_dir" / row["input_file_path"]
    return None
```

First-match policy — same as the tests use.

If the cache is absent (gitignored), every row becomes "skipped" with the regenerate command in the per-row log file. The script still produces an index (clearer than a hard exit; the index is the diagnostic).

## Loguru migration

### Add dependency

Edit [pyproject.toml](../pyproject.toml): add `loguru` to `[project].dependencies`. Run `uv pip install -e .` to refresh.

### One sink, plain format

Default sink: `stderr`, format `"<level> | <message>"` (no timestamp by default — these are mostly interactive CLI tools, the timestamp is noise; the per-converter file sink in `generate_report.py` *will* include timestamps, since those logs are read after the fact).

Configure once in a tiny `src/anndata_proteomics/_logging.py` helper:

```python
from loguru import logger
import sys

def configure_default_sink() -> None:
    """Idempotent default config: stderr, level=INFO, plain '<level> | <message>'."""
    logger.remove()
    logger.add(sys.stderr, format="{level} | {message}", level="INFO")
```

Each console-script entry point calls `configure_default_sink()` once at top of `main()`. Library modules just `from loguru import logger` and emit messages — they don't touch sink config.

## Concrete behavior changes — `tools/generate_report.py`

Current ([tools/generate_report.py](../tools/generate_report.py)):
- positional `data` arg → required input file
- runs one conversion
- rebuilds index from globbed `*.meta.json` (so old runs accumulate)

New:
- **No positional `data` arg.** Default invocation = "do all packaged converters".
- Iterates `iter_packaged_rules()`; per rule, looks up the input via `test_data.find_test_data(rule.software_name)`.
- **Per-converter scoped log sink:** at the start of each converter's run, `handler_id = logger.add("<output_dir>/<stem>.log", format="{time:HH:mm:ss} | {level:<7} | {message}", level="DEBUG")`. At the end (success, skip, or fail), `logger.remove(handler_id)`. The same file also captures Rscript stdout/stderr via `subprocess.run(..., stdout=fh, stderr=fh)` opened in append mode after the loguru sink writes its banner.
- Per rule outcomes:
  - **ok** — `<stem>.h5ad` + `<stem>.html` + `<stem>.meta.json` + `<stem>.log`
  - **skipped** — only `<stem>.meta.json` (status=skipped) + `<stem>.log` (with the reason, e.g. "no row with status=ok for software_name=PEAKS in raw_file_db_downloaded.csv"). `<stem>` for skipped uses `_sha8(software_name)` since there's no input path.
  - **failed** — same as skipped layout, plus the traceback in the log; `<stem>.h5ad` may or may not exist depending on where it failed (don't rely on it).
- At entry, when not filtered by `--rule`, clear `*.meta.json`, `*.h5ad`, `*.html`, `*.log`, `index.html` in `--output-dir`.
- Rebuild `index.html` with the 6-column schema; render skipped / failed rows explicitly.

### CLI surface

argparse stays:
- `--output-dir PATH` (default: `examples/results/`)
- `--rule SOFTWARE` (optional, repeatable) — restrict to specific `software_name`(s); useful for re-running one converter without nuking the rest. When passed, **do not** clear the output dir.
- `--log-level LEVEL` (optional, default `INFO`) — passed through to `configure_default_sink()` and the per-converter file sinks.

### Meta JSON schema

```json
{
  "status": "ok",                     // "ok" | "skipped" | "failed"
  "software": "DIA-NN",
  "stem": "diann_<sha8>",
  "input_path": "/.../input_file.txt",
  "h5ad_path": "diann_<sha8>.h5ad",   // present only when status=ok
  "html_path": "diann_<sha8>.html",   // present only when status=ok
  "log_path": "diann_<sha8>.log",     // always present
  "n_obs": 12345,                     // present only when status=ok
  "n_var": 6789,                      // present only when status=ok
  "layers": [{"name": "intensity", "n_obs": 12345, "n_var": 6789}, ...],
  "error": null                       // string when status != "ok", else null
}
```
